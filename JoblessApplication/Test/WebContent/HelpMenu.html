  • Installation and overview.
    JoblessDB
    I. Installation and System Requirements

    JoblessDB is a J2EE Web application for job advertisements / job search.

    Software requirements:


    Server: GlassFish 4.0
    Database: MySQL
    The application requires the set up of connection pool and JDBC resource.


    Libraries:
    JRE System Library [java-8-oracle]
    JSF 2.2 (Mojarra 2.2.0)
    MySQL JDBC Driver
    Web App Libraries
    guava-18.0

    II. Roles and levels of access:


    Online registration is available only for User and Company roles with different profile
    specifications required for the two types of users.
    Administrators' / accountants' profiles are available by default in the
    initial database. Owner information (necessary for the preparation of the invoices)
    is once added in the database and can be accessed by
    accountant(s) / administrator(s).

    - Guest:

    The guest user can view all job advertisements order by priority,
    search for advertisements (possible search keys: key word in the title, location, category, company name, VIP status),
    view advertisement, answer / send inquiry to the company.
    S/he can also apply for standard advertisements via fulfilling / submitting personal CV, writing /
    submitting Cover letter.

    - User:

    The user account inherits the basic functionality of the guest user. However, this is a registered user and can also
    apply for VIP advertisements by filling the tests, included in the VIP advertisement. User registration requires
    first and last name, valid e-mail, user name and password necessary for further login.
    Validation of the e-mail and check of user name availability is added as well as validation (not null and more than 5 symbols)
    and confirmation of the password. If invalid data (e-mail, user name or password) are provided,
    the registration form is rendered with corresponding messages output.
    After successful registration the user is invited to login in order to access the corresponding functionality.

    - Company:

    The company account is oriented towards job providers. It allows a registered company to submit a standard or VIP job advertisement.
    In order to access this functionality, the user has to be registered. Information required for successful registration is:
    company name, type (), address, valid e-mail, EIK (Единен идентификационен код; БУЛСТАТ), MOL (Материално отговорно лице),
    as well as user name and password necessary for login on the web-site. Similarly to the User registration form validation of user name,
    e-mail and password is added. After successful registration the company-user is invited to login in order to access
    the corresponding functionality.

    - Administrator (or admin):

    The admin accounts are initially set up in the database and can not be registered online. The admin uses a user name and
    password in order to access the admin panel. S/he can view new and old job advertisements, approve and publish new advertisements,
    remove old ones, change priority of the advertisements, update / delete Users, approve / update Company users.
    More information is provided in the admin panel.

    - Accountant:

    Similarly to the admin account, the accountant profiles are set in the database initially.
    In order to access the corresponding functionality, the accountant has to authenticate her/himself with account user name and password.
    The main functionality of the accountant user is the control over the payments connected with VIP advertisements and the
    preparation of the necessary documents. More information is provided in the accountant panel.

    - Owner:

    Information about the Owner is kept in the Database and can be accessed by th accountant for the
    preparation of the invoices for the paid VIP advertisements.

  • Login and Registration Help Tab

    Login and Registration Help Tab

    Find all the information needed to operate with login and registration module.

    MainPage

    JoblessDB application provides several options to its users as presented on the MainPage:

    - Login options:

    Four login options are available with respect to the different roles. In order to access User, Company, Administrator or Accountant modules,
    the user has to authenticate him/herself by providing username and password. While Administrators' and Accountants' credentials are
    initialized and kept in the database, User and Company types of users have to register online by following the corresponding links,
    provided on the MainPage (To register as user, click here. or To register as company, click here., respectively).
    Entered username & password are compared with the information in the database and if provided information is correct,
    the successful login redirects user to the required module.


    In order to be able to publish advertisements, Companies have to be approved by the Administrator who has the responsibility
    to verify the information provided by the Company at the registration.

    - “Continue as Guest” allows the user only to view all job advertisements.

    - Registration of User (To register as user, click here.)

    In order to register the User has to provide First and Last name, valid e-mail, username and password.
    Registration will be successful only if valid data are provided. Validation of the provided e-mail is performed during the typing process.
    Availability of the username (i.e. whether the desired username is not already used by another user) is also checked during the
    registration procedure. The validation of the password includes length (minimum 5 symbols) and confirmation of the entered password.
    White spaces are considered invalid input. Proper Error messages would appear if provided e-mail, username or
    password do not match the validation criteria.

    - Registration of Company (To register as company, click here.)

    In order to register the Company has to provide Company name, type, address, e-mail, EIK (единен идентификационен код),
    MOL(материално отговорно лице), username and password. Company type (ET, EAD, EOOD,...) can be chosen from a SelectOneMenu.
    In case no type is chosen, the type of the company-user will be set in the database as "Undefined".
    Similarly to the User registration, Company registration will be successful only if valid data are provided.
    Validation of the provided e-mail is performed during the typing process. Availability of the username
    (i.e. whether the desired username is not already used by another Company) is also checked during the registration procedure.
    The validation of the password includes length (minimum 5 symbols) and confirmation of the entered password.
    White spaces are considered invalid input. Proper Error messages would appear if provided e-mail, username or password
    do not match the validation criteria.


    No validation of the EIK is considered as this information is supposed to be verified by the Administrator.
    After successful registration the Company user is welcomed to login but (s)he will be able to publish advertisements only
    after the Company profile is approved by the Administrator.

  • Accounting Help Tab

    Accounting Help Tab

    Find all the information needed to operate with the Accounting module.

    Accounting Main page.

    On the main page there is one drop-down menu with filters, where under different criteria the user may look for invoices.
    When the user choose filter he has to press Submit button to continue to the chosen page. Also on the main page there is a button
    "Add new invoice" pressing which the user goes to page for adding new invoice.

    Show all invoices page

    The page shows all invoices in the database in data table. At the end of each column there is button “Full info” which takes
    you to page where you can see whole information about the current invoice. And there is button "Make paid" which sets the
    current invoice from unpaid to paid. At the end of the page there is button for "Export" to Excel table which shows message
    for success or failure, and also there is "Home" button which takes you to the Main page.

    Show all invoices by day page

    Accordingly to the preceding page this one shows invoices replying to the criteria taken from the input of the user
    for specific year, month and day. The page also provides the option for "Export" to Excel table and "Home" buttons.

    Show all invoices by month page

    Accordingly to the preceding page this one shows invoices replying to the criteria taken from the input of the
    user for specific year and month. The page also provides the option for "Export" to Excel table and "Home" buttons.

    Show all invoices by period page

    This page provides to the user option to search for a invoice for a distinct period of time.
    The start period and the end period of the search is taken from the user input.

    Show all invoices by company name page

    In this page the user can load all invoices for a distinct company chosen from the drop-down menu
    which automatically loads the invoices for the company.

    The Is paid page

    This page shows all waiting payment invoices.

    The Due payment page

    The page provides information about all late invoices( there is option for the client to pay the invoice in 5 days).
    The page automatically calculates the period for the user.

  • Users And Companies Help Tab

    Users And Companies Help Tab

    Find all the information needed to operate with the User and Company Modules

    -Companies

    Companies can create new ads. Each ad can be defined as Regular, VIP or VIP with test.

    Regular advertisements

    Those are free of charge. They do not have to possibility to have a test.
    All guests can apply to them via sending an email to the company.
    This ad will remain active for five up to thirty days.

    - VIP ADS

    Those advertisements are paid. Guests can see and make an inquiry about the AD. Only logged users can apply
    via writing and sending CV and Cover letter. This ad will remain active for five up to thirty days.

    VIP ads with test

    Those advertisements are paid. Companies can create a test. Guests can see and make an inquiry about the AD.
    Only logged users can apply via taking the test and/or writing and sending CV and Cover letter.
    This ad will remain active for five up to thirty days.

    -Users

    Users of the company can be logged or guests

    - Guests

    Have to possibility to see all the currently active ads. Can apply to ads via sending an email to the company.

    - Logged Users

    Have to possibility to see all the currently active ads. Can apply to all ads by writing and sending CV and cover letter.
    Can take test if the advertisement has one. Applying to the ad, will send the relevant data automatically.

  • Administration Help Tab

    Find all the information needed to operate with Administration module.

    - Administration Panel options

    Tree options are available for redirecting to other panels AdvertismentPanel, CompanyPanel and UserPanel.

    -First option AdvertismentPanel

    When the first one, called “See all Ads”, is clicked it evokes event that displays a table with ads. Every ad has four buttons – update, publish, delete and make VIP.When pressed update it evokes event that open ".xhtml" file that have fields for updating the Ad and suitable button. Upon pressing it, a database connection is ensured and new values for the corresponding Advertisment are set. The second button is “publish”. Every ad has validation time. The expiration date is calculated by taking the activity days and adding the current date plus one more day. This information is set in the field “Expired”. Also the field “Approved” becomes “true”. The button “delete” erase the ads. When clicked, it evokes event that delete the ad through its id. Advertisment with expired date can be checked through “Expired Ads” if there are Expired then make the field "isExpired" to "true", located at the bottom of the screen.

    -Second option CompanyPanel

    All information for the companies is shown. When pressed update it evokes event that open ".xhtml" file that have fields for updating the CompanyProfile and suitable button. Upon pressing it, a database connection is ensured and new values for the corresponding CompanyProfile. Company existence should be checked. There are two possibilities for that – 1 for approval and 2 for disapproval. When the status is 1, then the company is legal and the company can publish ads. If the company is checked and the administrator has not confirmed the existence of ad, the status becomes 2 and the “disapproved” button should be clicked so the company can not publish ads.

    -Third option UserPanel

    All information for the users is shown.When pressed update it evokes event that open ".xhtml" file that have fields for updating the UserProfile and suitable button. Upon pressing it, a database connection is ensured and new values for the corresponding UserProfile are set.
